# 🔄 Force Railway to Use Latest Code

## The Problem
Railway is building OLD code even though GitHub has the fixed code.

## Solution: Force Redeploy

### Method 1: Trigger New Deployment in Railway Dashboard

1. Go to your Railway project dashboard
2. Click on **auth-service**
3. Go to **Settings** tab
4. Scroll down to **Danger Zone**
5. Click **"Redeploy"** or **"Trigger Deploy"**
6. Railway will pull fresh code from GitHub

### Method 2: Check Railway Build Settings

1. Go to **auth-service** in Railway
2. Click **Settings**
3. Check **"Source"** section:
   - Branch should be: `master`
   - Root Directory should be: `backend/`
   - Build Command should be: `cargo build --release --package auth-service`
   - Start Command should be: `./target/release/auth-service`

### Method 3: Make a Dummy Commit to Force Rebuild

If Railway still uses old code, make a small change:

```bash
cd Nexus-storage

# Add a comment to force new commit
echo "# Updated" >> backend/services/auth-service/README.md

git add .
git commit -m "Force Railway redeploy"
git push origin master
```

Railway will detect the new commit and rebuild.

### Method 4: Check Railway Environment Variables

Make sure these are set in Railway for auth-service:

```
DATABASE_URL=<auto-generated by Railway Postgres>
REDIS_URL=<auto-generated by Railway Redis>
JWT_SECRET=your-secret-key
PORT=8080
```

## Verify the Fix

After redeploying, check the build logs. You should see:
```
✅ Compiling auth-service v0.1.0
✅ Finished `release` profile [optimized]
```

NO MORE ERRORS about:
- ❌ "set `DATABASE_URL` to use query macros"
- ❌ "failed to resolve: chrono"

## Still Failing?

If it still fails, Railway might be using a cached layer. Try:
1. Delete the service in Railway
2. Create a new service
3. Connect to GitHub repo
4. Set root directory to `backend/`
5. Deploy fresh
